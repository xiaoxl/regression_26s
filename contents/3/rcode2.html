11  R code

How to use lm function.

x1 <- runif(100)
x2 <- runif(100)
eps <- rnorm(100, 0, 0.5)
y <- 1+2*x1+3*x2+eps
model <- lm(y~x1+x2)
  1. Model coefficients
model$coefficients
model$coefficients["(Intercept)"]
model$coefficients["x1"]
model$coefficients["x2"]

When using SLR, we could directly read the coefficients and plot lines.

plot(x1, y)
abline(lm(y ~ x1)$coefficients)
  1. Some computed values

We could directly find the residuals and the fitted values.

model$residuals
model$fitted.values
  1. Model properties The standard error, \(R^2\), \(R_a^2\), and \(F\)-statistic can be gotten from here.
summary(model)$sigma
summary(model)$r.squared
summary(model)$adj.r.squared
summary(model)$fstatistic

The individual Std. error and t value and p value can also be read directly.

summary(model)$coefficients
summary(model)$coefficients['x2', 't value']
  1. Why Does summary(lm()) Show t-tests Instead of F-tests? Regression focus:

summary(lm()) emphasizes coefficient estimates (slopes), so t-tests are more intuitive.

It answers: “Is this specific predictor’s slope ≠ 0?”

ANOVA focus:

anova(lm()) emphasizes variance explained by each term.

It answers: “Does this predictor (or model) explain significant variance?”

  1. Key Takeaways For simple regression (1 predictor):

The t-test (from summary) and F-test (from anova) are identical.

For multiple predictors:

t-tests assess individual slopes (adjusting for other predictors).

F-tests assess groups of predictors (e.g., all levels of a categorical variable).

ANOVA is a generalization:

t-tests are a special case of ANOVA when testing a single predictor.

12 A Concrete Example for Formula Language and ANOVA

In this section, we use one concrete example to connect the following topics:

  • formula language
  • global \(F\)-test
  • individual \(t\)-tests
  • Type I, Type II, and Type III ANOVA
  • interaction terms
  • higher-degree terms
  • centering
  • multicollinearity

The goal is to see how all of these ideas fit together in one regression analysis.

12.1 Example setup

Suppose we want to study how an exam score depends on:

  • hours: study hours
  • attend: attendance rate
  • tutor: whether the student received tutoring (0 = no, 1 = yes)

A reasonable model might include:

  1. main effects of all predictors
  2. an interaction between hours and attend
  3. a quadratic term for hours to capture diminishing returns

So we consider the model

\[ y=\beta_0+\beta_1x_1+\beta_2x_2+\beta_3x_3+\beta_4x_1x_2+\beta_5x_1^2+\varepsilon \]

where

  • \(y=\) score
  • \(x_1=\) hours
  • \(x_2=\) attendance
  • \(x_3=\) tutoring indicator

In words:

  • more hours may improve the score
  • better attendance may improve the score
  • tutoring may improve the score
  • the effect of hours may depend on attendance
  • the effect of hours may not be linear

12.2 Formula language

In R formula language, the above model can be written as

score ~ hours + attend + tutor + hours:attend + I(hours^2)
score ~ hours*attend + tutor + I(hours^2)
hours + attend + hours:attend
Note

Formula interpretation

hours + attend + tutor adds main effects

hours:attend adds only the interaction

hours*attend adds both main effects and the interaction

I(hours^2) tells R to treat hours^2 as arithmetic, not formula syntax

set.seed(1)

n <- 120
df <- data.frame(
    hours = runif(n, 0, 10),
    attend = runif(n, 0.6, 1.0),
    tutor = rbinom(n, 1, 0.4)
)

df$score <- 50 +
    4 * df$hours +
    15 * df$attend +
    6 * df$tutor +
    2 * df$hours * df$attend -
    0.35 * df$hours^2 +
    rnorm(n, 0, 5)

The quadratic term \(h^2\) introduces curvature.

\[ h^*=(b1+b4a)/2b5 \]

for a fixed attendance level \(a\).

because hours*attend automatically expands to

hours + attend + hours:attend

Equivalently, \[ \beta_4=\frac{\partial^2E(y)}{\partial h,\partial a}. \]

\[ h^*=-\frac{\beta_1+\beta_4a}{2\beta_5} \]

for a fixed attendance level \(a\).

15 Global F-test and individual t-tests

15.1 Global F-test

For a multiple regression model, the global \(F\)-test examines

  • \(H_0\): all slope coefficients are \(0\)
  • \(H_a\): at least one slope coefficient is not \(0\)

For m3, this is the test of whether the model as a whole explains variation in score.

anova(m3)
## Analysis of Variance Table
## 
## Response: score
##               Df Sum Sq Mean Sq  F value    Pr(>F)    
## hours          1 3559.2  3559.2 134.6217 < 2.2e-16 ***
## attend         1  892.4   892.4  33.7551 5.767e-08 ***
## tutor          1 1185.2  1185.2  44.8297 8.485e-10 ***
## I(hours^2)     1  808.2   808.2  30.5677 2.072e-07 ***
## hours:attend   1   52.2    52.2   1.9735    0.1628    
## Residuals    114 3014.0    26.4                       
## ---
## Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1

The global test answers the question:

Does this regression model provide useful information about the response?

15.2 Individual t-tests

The summary() output gives individual \(t\)-tests for each coefficient.

For example, the test for the interaction term is

\[ H_0:\beta_4=0 \qquad\text{vs}\qquad H_a:\beta_4\ne 0. \]

The test for the quadratic term is

\[ H_0:\beta_5=0 \qquad\text{vs}\qquad H_a:\beta_5\ne 0. \]

These tests help us decide whether each term contributes after accounting for the others in the fitted model.

16 ANOVA tables and their meanings

In multiple regression, there is more than one way to define the contribution of each term. This leads to different ANOVA types.

16.1 Type I ANOVA

Type I ANOVA is sequential. Terms are tested in the order they enter the model.

anova(m3)
## Analysis of Variance Table
## 
## Response: score
##               Df Sum Sq Mean Sq  F value    Pr(>F)    
## hours          1 3559.2  3559.2 134.6217 < 2.2e-16 ***
## attend         1  892.4   892.4  33.7551 5.767e-08 ***
## tutor          1 1185.2  1185.2  44.8297 8.485e-10 ***
## I(hours^2)     1  808.2   808.2  30.5677 2.072e-07 ***
## hours:attend   1   52.2    52.2   1.9735    0.1628    
## Residuals    114 3014.0    26.4                       
## ---
## Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1

For the formula

score ~ hours*attend + tutor + I(hours^2)

the terms are entered in the order:

  1. hours
  2. attend
  3. tutor
  4. hours:attend
  5. I(hours^2)

So Type I tests:

  • hours first
  • then attend after hours
  • then tutor after hours and attend
  • and so on
CautionType I depends on order

If you change the order of predictors in the formula, the Type I ANOVA table changes.

For example:

m3_alt <- lm(score ~ tutor + attend + hours + hours:attend + I(hours^2), data = df)
anova(m3_alt)
## Analysis of Variance Table
## 
## Response: score
##               Df Sum Sq Mean Sq  F value    Pr(>F)    
## tutor          1  783.1   783.1  29.6196 3.050e-07 ***
## attend         1  936.8   936.8  35.4339 2.977e-08 ***
## hours          1 3917.0  3917.0 148.1531 < 2.2e-16 ***
## I(hours^2)     1  808.2   808.2  30.5677 2.072e-07 ***
## attend:hours   1   52.2    52.2   1.9735    0.1628    
## Residuals    114 3014.0    26.4                       
## ---
## Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1

This ANOVA table will generally differ from the previous one.

16.2 Type II ANOVA

Type II ANOVA tests each main effect after adjusting for the other main effects, but not after adjusting for higher-order relatives containing that term.

library(car)
Anova(m3, type = 2)
## Anova Table (Type II tests)
## 
## Response: score
##               Sum Sq  Df F value    Pr(>F)    
## hours        1824.65   1 69.0150 2.292e-13 ***
## attend        595.21   1 22.5128 6.100e-06 ***
## tutor        1124.39   1 42.5283 1.985e-09 ***
## I(hours^2)    848.20   1 32.0818 1.124e-07 ***
## hours:attend   52.18   1  1.9735    0.1628    
## Residuals    3013.99 114                      
## ---
## Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1

Roughly speaking, Type II asks:

  • does hours matter after accounting for attend and tutor?
  • does attend matter after accounting for hours and tutor?
  • does hours:attend matter after accounting for the main effects?
  • does I(hours^2) matter after accounting for the lower-order terms?

Type II is often a good default when the model respects the hierarchy principle.

NoteHierarchy principle

If an interaction term or polynomial term is in the model, the related lower-order terms should also remain in the model.

Type III ANOVA tests each term after adjusting for all other terms in the full model.

For example, the Type III test for hours is essentially a test of

\[ H_0:\beta_1=0 \]

in the full model containing

  • attend
  • tutor
  • hours:attend
  • I(hours^2)

This often causes interpretation problems, because with interaction present, the coefficient \(\beta_1\) no longer represents a universal main effect.

In fact, in the interaction model,

\[ \frac{\partial E(y)}{\partial h}=\beta_1+\beta_4a+2\beta_5h, \]

so \(\beta_1\) corresponds to a slope evaluated at specific reference values.

17 A concrete interpretation of Type I, II, and III

Using m3, the three ANOVA types answer different questions.

17.1 Type I question

If we add terms one by one in this exact order, how much extra variation does each term explain?

So Type I is order-based.

17.2 Type II question

Does this term help explain the response, after accounting for other comparable lower-order terms?

So Type II is usually closer to the idea of testing the practical contribution of a term.

17.3 Type III question

Is this coefficient zero in the full model, after everything else is already included?

So Type III is coefficient-based in the full model.

NotePractical summary
  • Type I: sequential, order dependent
  • Type II: usually best for well-structured regression models
  • Type III: tests coefficients in the full model; often hard to interpret unless predictors are centered and contrasts are chosen carefully

18 Why Type II and Type III differ when interaction exists

Consider the interaction model

\[ E(y)=\beta_0+\beta_1h+\beta_2a+\beta_3ha. \]

Then

\[ \frac{\partial E(y)}{\partial h}=\beta_1+\beta_3a. \]

So the coefficient \(\beta_1\) is the effect of hours only when \(a=0\).

If attend = 0 is not meaningful, then the Type III test of \(\beta_1=0\) is not testing a useful scientific question.

By contrast, Type II usually asks whether hours contributes in a broader model-comparison sense.

This is why Type II and Type III may disagree when interaction is present.

20 Multicollinearity and VIF

Polynomial and interaction terms often create multicollinearity.

For example:

  • hours and hours^2 may be strongly correlated
  • hours, attend, and hours:attend may also be related

We can check this with VIF.

20.1 VIF for the uncentered model

vif(m3)
## there are higher-order terms (interactions) in this model
## consider setting type = 'predictor'; see ?vif
##        hours       attend        tutor   I(hours^2) hours:attend 
##    68.406748     4.432928     1.014656    17.654191    63.815488

20.2 VIF for the centered model

vif(m4)
## there are higher-order terms (interactions) in this model
## consider setting type = 'predictor'; see ?vif
##       hc       ac    tutor  I(hc^2)    hc:ac 
## 1.037259 1.040772 1.014656 1.046106 1.061252

Usually, centering reduces the collinearity between lower-order and higher-order terms.

NoteInterpretation of VIF

A large VIF means that the corresponding predictor is highly explained by other predictors.

This inflates the variance of its estimated coefficient and may make a truly important term appear non-significant.

21 A nested-model view of ANOVA tests

It is useful to understand ANOVA tests through nested model comparisons.

21.1 Testing the interaction

To test whether the interaction is needed, compare

  • reduced model: \[ \text{score} \sim \text{hours} + \text{attend} + \text{tutor} + I(\text{hours}^2) \]

  • full model: \[ \text{score} \sim \text{hours} + \text{attend} + \text{tutor} + I(\text{hours}^2) + \text{hours}:\text{attend} \]

In R:

m_no_int <- lm(score ~ hours + attend + tutor + I(hours^2), data = df)
anova(m_no_int, m3)
## Analysis of Variance Table

22 Recommended teaching workflow

A practical workflow for students is:

  1. Start with a scientifically reasonable full model.
  2. Use the global \(F\)-test to verify that the model is useful.
  3. Use a Type II ANOVA table to assess terms.
  4. Interpret interaction terms before interpreting main effects.
  5. Check for multicollinearity using VIF.
  6. Center quantitative predictors when interactions or polynomial terms are present.
  7. Refit and interpret the coefficients again.
  8. Use nested model comparisons when testing whether a specific higher-order term is needed.

24 Short teaching summary

NoteOne-paragraph summary

In multiple linear regression, formula language describes which main effects, interaction terms, and higher-degree terms are included in the model. Once the model is fit, the global \(F\)-test checks whether the model is useful overall, while individual \(t\)-tests examine specific coefficients. In multiple regression, ANOVA is not unique: Type I is sequential and depends on order, Type II is usually preferred for hierarchical models, and Type III tests coefficients in the full model and is most meaningful after centering. Interaction terms change the interpretation of main effects, and polynomial terms often create multicollinearity, so centering and VIF checks are often important parts of the analysis.
